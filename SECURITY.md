# Security policy

## Supported versions

The project is pre-1.0. Only the newest tagged release will receive security fixes once public releases begin.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability, leaked signing material, unsafe ADB behavior, or endpoint-association flaw. Use GitHub’s private vulnerability reporting for the repository. If that feature is not yet enabled, contact the repository owner privately through the contact method on their GitHub profile.

Include the affected version, connection route, exact reproduction steps with secrets and pairing codes removed, and whether a physical Android device or only a fixture was involved.

Never include live pairing codes, ADB private keys, device recordings, signing certificates, notarization credentials, or App Store Connect API keys in a report attachment.
